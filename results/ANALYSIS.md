# 실험 결과 상세 분석 — HAN+GTN+PDGNN, 14개 데이터셋 × A~D

test Macro-F1, **mean±std (seed 0/1/2)**. 수치 근거는 [`SUMMARY.md`](SUMMARY.md), 원본은
`results/campaign/`. 성능 주장이 아니라 실측 기반 해석이며, 안 되는 케이스도 그대로 적는다.

---

## 0. TL;DR

1. **위상(GTN+PDGNN)의 순이득(Δ=C2−A1)은 조건부다.** node feature 가 약하고(headroom),
   메타패스가 클래스-동질적이며(homophily), 그래프가 실제 연결되고(connectivity), 단일라벨일 때만
   순이득이 난다 — DBLP +0.081, AMiner +0.070, AIFB +0.196 (상대 +10~45%).
2. **"위상을 쓰는 것"과 "이득이 되는 것"은 다르다.** permutation 진단으로 보면 위상은 feature 강한
   곳에서도 *사용*되지만(ACM permΔ +0.064, dblp_pyg +0.037), 그 경우 순이득은 ~0 — **중복**이기 때문.
3. **과장 금지:** 전체 파이프라인(C2)이 plain GTN(A3)을 항상 이기지 못한다 — DBLP/PubMed/MUTAG 는
   GTN 단독이 더 높다.
4. **GTN 자동 메타패스 > random 은 보편적이지 않다** — DBLP/AIFB 는 그렇지만 AMiner 는 random 이 근소 우위.
5. **실패 유형 3가지**: featureless RDF 붕괴(MUTAG/BGS/AM), 초저성능 collapse(MAG/Freebase/PubMed),
   멀티라벨 무반응(IMDB/Yelp).

## 1. 실험 설계 (A~D)

| 코드 | 의미 | 구현 |

## 3. 주결과 — 위상의 순이득은 "node feature 여유" 가 결정

C2 가 A1 을 의미 있게 넘는 곳: **AIFB(+0.196), DBLP(+0.081), AMiner(+0.070)**. 셋 다 baseline 이
낮거나(0.43~0.44) 중간(0.79)이라 **여유(headroom)** 가 크다. 반대로 baseline 이 포화된 ACM(0.89)·
dblp_pyg(0.93)·IMDB 는 Δ≈0.

> **결정적 대비 — DBLP vs dblp_pyg**: 같은 저자분류 도메인, 같은 334차원 특징인데
> HGB-DBLP(baseline 0.793) → 위상 +0.081, MAGNN-DBLP(baseline 0.932) → −0.001.
> 효용을 가르는 건 도메인이 아니라 **baseline 의 여유**다. node feature 가 이미 충분하면 위상은 중복.

상대이득(relΔ)으로 보면 feature 가 약할수록 비율이 크다: AIFB +45%, AMiner +16%, DBLP +10%
(MAG +180% 는 절대값이 0.01 수준이라 해석 보류).

## 4. "위상을 쓰는가" vs "이득이 되는가" — permutation 이 분리해 보여줌

permΔ = C2 점수 − (test 때 위상 행을 섞었을 때 점수). **위상을 실제로 쓰면 섞을 때 떨어진다.**

- **위상을 강하게 사용**: AMiner +0.381, DBLP +0.260, AIFB +0.248. (사용 + 이득 둘 다)
- **사용은 하나 이득은 0 (중복)**: ACM +0.064, dblp_pyg +0.037 — 위상을 쓰긴 쓰는데 feature 와
  겹쳐 순이득 ~0.
- **아예 안 씀**: imdb_pyg/imdb/freebase/pubmed permΔ ≈ 0 — 섞어도 안 떨어짐 = 위상 미사용.

→ **핵심 통찰**: imdb_pyg 의 Δ+0.014 는 permΔ=0 이므로 *위상에서 온 게 아니다*(추가 파라미터의
노이즈). 즉 **진짜 위상 순이득은 DBLP·AMiner·AIFB 셋**으로 좁혀진다. ACM·dblp_pyg 는 "위상을 쓰지만
중복", 나머지는 "위상 미사용/붕괴".

## 5. 위상이 도움된 데이터의 공통점 (정량 + 체크리스트)

구조 속성 (homo = 메타패스 엣지 양끝 동일 클래스 비율; homoR = homo×#cls = 무작위 대비 배수;
mp_deg = 라벨 노드 메타패스 평균차수):

| 데이터 | A1(여유) | homo | homoR | mp_deg | 단일라벨 | 붕괴아님 | → 순이득 |
|--------|---------|------|-------|--------|---------|---------|---------|
| dblp | 0.79 ✅ | 0.50 | 2.0× ✅ | 20.7 ✅ | ✅ | ✅ | **+0.081** |
| aminer | 0.43 ✅ | 0.65 | 5.2× ✅ | 20.3 ✅ | ✅ | ✅ | **+0.070** |
| aifb | 0.44 ✅ | (관계 동질) | — | 11.0 ✅ | ✅ | ✅ | **+0.196** |
| acm | 0.89 ❌포화 | 0.71 | 2.1× ✅ | 21.9 ✅ | ✅ | ✅ | +0.004 |
| dblp_pyg | 0.93 ❌포화 | 0.80 | 3.2× ✅ | 20.7 ✅ | ✅ | ✅ | −0.001 |
| pubmed | 0.09 ✅ | **0.14** | **1.1×** ❌ | 12.6 ✅ | ✅ | ❌붕괴 | −0.027 |
| freebase | 0.14 ✅ | 0.60 | 4.2× ✅ | 13.6 ✅ | ✅ | ❌붕괴 | +0.003 |
| mutag/bgs/am | ~0.4 ✅ | — | — | **0.5** ❌ | ✅ | ❌단절 | 0.000 |
| imdb/yelp | ~0.3 | — | — | — | ❌멀티 | — | ≤0 |

**순이득(Δ>0.05)을 내려면 4가지가 동시에 필요:**
1. **헤드룸** — baseline 미포화 (ACM/dblp_pyg 는 이걸로 탈락).
2. **메타패스 동질성** — homoR ≫ 1 (PubMed 1.1×≈무작위로 탈락, 위상이 오히려 방해).
3. **연결성** — mp_deg 충분 (MUTAG/BGS/AM 은 0.5 단절로 탈락 → 위상이 빔).
4. **단일라벨 + 비붕괴** — 멀티라벨(IMDB/Yelp) 무반응, 초저성능 붕괴(MAG/Freebase) 무효.

각 비-순이득 데이터가 **정확히 한 조건씩** 위반한다는 점이 이 4조건의 타당성을 뒷받침한다.
거칠게: **Δ ∝ 헤드룸 × 동질성-세기** (둘 다 큰 DBLP/AMiner/AIFB 가 크게 이득). 직관 — PDGNN 은 ego 의
연결 구조를 요약하므로, 그 구조가 클래스와 정렬돼야(동질·연결) 신호가 되고, feature 가 이미 그 정보를
담으면(포화) 중복이 된다.

## 6. GTN(C2) vs 수동(B2) vs 무작위(D5) — 자동화는 부분적으로만 우위

위상이 작동하는 데이터 한정:
- **DBLP**: C2 0.874 > D5 0.837 > B2 0.806 — GTN 학습이 무작위·수동 모두 명확히 능가.
- **AIFB**: C2 0.634 > B2 0.618 > D5 0.590 — GTN > 수동 > 무작위.
- **AMiner**: D5 0.507 ≳ C2 0.495 > B2 0.474 — **무작위가 근소 우위**(GTN 의 학습 이점 없음).
- **ACM**: C2 0.892 ≈ B2 0.893 > D5 0.883 — 수동과 비등.

→ "GTN 자동 메타패스가 항상 더 낫다" 는 **과주장**. DBLP/AIFB 에선 분명하나 AMiner 에선 무작위 조합으로
충분하다. 플랜의 자동화 가설은 *일부* 데이터에서만 지지된다.

## 7. 전체 파이프라인(C2) vs plain GTN(A3) — 항상 이기지 못함 (정직)

A3 = GTN 의 자체 GCN 분류기(EPD·HAN 없음), 그 자체로 강한 이종 GNN. 단일라벨 비교:
- **GTN 단독이 더 높음**: DBLP 0.919 vs 0.874, MUTAG 0.650 vs 0.398, PubMed **0.395 vs 0.059**.
- **위상 파이프라인이 더 높음**: AMiner 0.495 vs 0.260, AIFB 0.634 vs 0.251.
- 비등: ACM(0.894/0.892), MAG, BGS.

특히 **PubMed**: plain GTN 0.395 인데 C2 0.059 로 붕괴 — 데이터가 아니라 **HAN+메타패스 단계**가
희소 생의학 그래프(라벨 5%)에서 무너지는 게 원인. **DBLP** 도 plain GTN 이 최고 — 위상 파이프라인을
얹는 게 손해. (A3 는 GTN 헤드가 CE 라 멀티라벨 imdb/freebase/am 에서 0 으로 degenerate → 멀티라벨
비교는 제외.) → **"위상 파이프라인이 항상 최선"은 거짓.**

## 8. Ablation — 민감한 건 kNN 뿐

- **D1 (kNN 제거)**: ego 폭증으로 >2h 미완 → 성능 이전에 **실행 가능성의 필수 조건**.
- **D2 (MIN 집계 제거)**: 거의 모두 C2 와 동등하거나 약간 높음 (ACM 0.898 vs 0.892, Freebase
  0.158 vs 0.143) → **PDGNN 의 MIN 집계는 기여 없음**(SUM-only 로 충분).
- **D3 (채널 2/8)·D4 (깊이 1/3)**: 대부분 차이 < std (평탄). 채널 4·깊이 2가 약간 안정적 정도.

→ 위상 파이프라인의 결과는 집계방식/용량에 둔감하고, **그래프 희소화(kNN)** 에만 민감.

## 9. 실패 유형 분석 (3가지)

1. **featureless RDF 붕괴 (MUTAG/BGS/AM)**: 모든 설정 동일·std 0 → 다수 클래스만 예측. 원인 =
   one-hot 특징 + 라벨 sparse(4~17%) + 메타패스가 라벨 노드를 거의 연결 못 함(mp_deg 0.5).
   위상은커녕 분류 자체가 학습 안 됨. (예외: AIFB 만 RDF 중 성공 — 관계가 affiliation 으로 강하게
   클래스 정렬.)
2. **초저성능 collapse (MAG/Freebase/PubMed)**: 절대값 0.01~0.14. 349클래스·subsample 소형 test(MAG),
   featureless+5~14% 라벨(Freebase/PubMed). 모델이 거의 majority 로 수렴 → 위상이 실릴 여지 없음
   (permΔ≈0).
3. **멀티라벨 무반응 (IMDB/Yelp)**: BCE 로 도는데 위상이 순이득 없음(permΔ≈0 또는 음수). 멀티라벨은
   per-node 위상 PI 가 라벨 집합과 정렬되기 어려움.

## 10. 강건성(분산) — "위상=robustness" 는 일관되지 않음

- featureless/저성능에서 분산 큼: AMiner ±0.08~0.10, AIFB ±0.10, MAG ±0.02(절대값 대비 큼).
- feature 강한 곳은 안정: ACM/dblp_pyg ±0.00~0.01.
- 선행 DBLP 연구의 "위상이 ~46% 분산 감소" 패턴은 본 14개에서 **재현되지 않음** — 오히려 위상이
  의미 있는 featureless 데이터에서 분산이 더 크다.

## 11. 한계 & 위협 요인 (threats to validity)

- **subsample**: 대형(AMiner/MAG/RCDD류/RDF/HNE)은 라벨 중심 연결 부분그래프 ~6K 로 축소 →
  절대 수치는 full-graph 와 다를 수 있음(특히 MAG test 작아 노이즈). 단 A1/C2 가 같은 부분그래프라
  *상대비교(Δ)* 는 유효.
- **PI 재구현**: 원본 cython sg2dgm 가 아닌 numpy 표준 PI(범위/σ 다름) — 수치 비재현, 단 전 데이터셋
  동일 설정이라 내부 비교는 일관.
- **seed 3개**: 분산 주장 통계력 약함.
- **GTN target-restricted + CE 헤드**: 멀티라벨 A3 무의미; 전체 노드집합 원본 GTN 과 다름(문서화된 적응).
- **staged 학습(C4 미구현)**: task loss 가 GTN 까지 역전파 안 됨.

## 12. 실무 가이드 & 후속

**언제 위상(GTN+PDGNN)을 쓸 가치가 있나** (체크리스트):
- [ ] node feature 만으로의 baseline 이 포화가 아님(헤드룸 有)
- [ ] 메타패스가 클래스-동질적(같은 클래스끼리 연결; homoR≫1)
- [ ] 메타패스 그래프가 충분히 연결(고립 라벨 노드 적음)
- [ ] 단일라벨, 모델이 majority 로 붕괴하지 않음

넷 다 ✅ → 위상이 유의미한 이득(예: DBLP/AMiner/AIFB). 하나라도 ❌ → 위상 생략이 합리적.

**후속 방향** (관측에서 직접 도출):
1. **gating fusion** — feature 강한 곳에서 위상 채널을 자동으로 끄기(ACM/dblp_pyg 의 중복 제거).
2. **멀티라벨 대응** — GTN 헤드/위상 융합을 BCE·라벨집합 인지로(IMDB/Yelp).
3. **희소 그래프 보강** — PubMed/RDF 처럼 라벨 sparse 한 곳의 메타패스 연결성 강화(또는 위상 미적용 판단).
4. **end-to-end(C4)** — 미분가능 EPD 로 task loss 를 GTN 까지(현재 staged).
